C.2 Create Payment Link
const Razorpay = require('razorpay');const razorpay = new Razorpay({  key_id: process.env.RAZORPAY_KEY_ID,  key_secret: process.env.RAZORPAY_KEY_SECRET,});async function createPaymentLink(orderId, amount, customerPhone, customerName) {  const paymentLink = await razorpay.paymentLink.create({    amount: amount * 100, // Razorpay takes paise    currency: 'INR',    description: `Ironing Service - Order #${orderId}`,    customer: { name: customerName, contact: customerPhone },    notify: { sms: false, email: false }, // sent via WhatsApp ourselves    callback_url: `https://yourdomain.com/payment-success`,    callback_method: 'get',  });  return paymentLink.short_url; // e.g. https://rzp.io/l/abc123}

PART D — V2 Critical Logic Changes
D.1 Partner Login — ID + Password (Backend)
routes/auth.js — POST /api/auth/partner/login
Receive username + password
Find partner by username in DB
bcrypt.compare(password, partner.password_hash)
If match → issue JWT with { partnerId, role: 'partner' }
If no match → return 401 Unauthorized
middleware/auth.js — protect partner routes
Verify JWT → check role === 'partner'
Add req.partnerId = token.partnerId
GET /api/partner/jobs → WHERE partner_id = req.partnerId (partner sees only their own jobs)
D.2 Cloth Check Before Pre-Payment — Corrected Flow
Step
Old V1 Order
New V2 Order (Correct)

Why this order is correct: cloth condition must be documented and accepted by the customer BEFORE the count and payment. Disputes raised after payment are harder to resolve — checking first creates a clean record before any money changes hands.

PART J — WhatsApp Template Creation (Exact Steps)
Templates are pre-approved message formats — needed ONLY when your business messages the customer FIRST, outside an active 24-hour service window. Customer-initiated flows (Hi, Place Order, menu buttons) need zero template approval.
J.1 How to Create a Template
Step 1: Go to developers.facebook.com → your Ironing Service app
Step 2: Left sidebar → WhatsApp → Message Templates
Step 3: Click 'Create Template'
Step 4: Category = UTILITY (not Marketing — wrong category = higher cost or rejection)
Step 5: Template Name = lowercase + underscores only, e.g. payment_request (cannot change after approval)
Step 6: Language = English (en_US); add Tamil later as a separate template if needed
Step 7: Body = message text with {{1}}, {{2}}… variables, max 1024 characters
Step 8: Footer (optional, max 60 chars) and Buttons (optional, max 3) if needed
Step 9: Submit for review — Meta approval typically takes minutes to a few hours
